Error Handling Protocol

Graceful degradation designed for production environments.

To prevent application crashes on your frontend, our API responds to quota limits with an HTTP 200 OK. The error message is injected directly into the standard response payload, ensuring your end-users receive a seamless notification rather than an application-breaking 400 or 429 status code.

{
  "id": "chatcmpl-error",
  "object": "chat.completion",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "⚠️ The response is too long. Please ask a shorter question or break it into smaller parts."
    }
  }]
}
